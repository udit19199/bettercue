# Product

## Register

product

## Users

Developers, prompt engineers, and AI users who write prompts regularly. They work in fast iteration cycles — prototyping, testing, refining — and need a frictionless tool to sharpen their prompts across multiple LLM providers without leaving the browser.

## Product Purpose

bettercue takes a raw prompt and rewrites it through an LLM to be more effective, clear, and structured. The Chrome extension brings this to where users work: the browser. Multi-provider (Ollama, OpenAI, Anthropic, Google), with preset rewrite styles and clarifying questions. Success is a prompt that performs better on first use, with zero tool friction.

## Brand Personality

Calm · Clear · Capable

Voice is precise and unobtrusive — a capable expert, not a hype machine. No marketing fluff, no sci-fi posturing. The interface should feel like a well-made tool: it gets out of your way when you know what you're doing, and guides subtly when you don't.

## Anti-references

Avoid all generic AI tool tropes:
- No chatbot bubbles, no "AI assistant" persona
- No sci-fi motifs, no robot icons, no neural network visuals
- No glowing circuit boards or "futuristic" gradient meshes
- No heavy enterprise SaaS patterns (blue navbars, crowded dashboards)

The tool is practical, not a spectacle.

## Design Principles

1. **Practice what you preach** — The tool optimizes prompts for clarity and structure; the UI itself should be equally clear and well-structured. Every design decision earns its place.

2. **Calm confidence** — Restrained, not loud. The interface trusts the output to speak for itself. Motion is considered but quiet. Color is purposeful but not decorative.

3. **Developer empathy** — Built for people who work with language models: fast, transparent about what's happening, keyboard-friendly, and honest about limits. Show token counts, provider info, and processing state without ceremony.

4. **Substance over decoration** — No chatbot bubbles, no sci-fi gradients, no robot motifs. The aesthetic is a well-designed tool: precise spacing, intentional typography, considered contrast. Every pixel serves the task.

## Accessibility & Inclusion

Standard good practice: reasonable contrast (≥4.5:1 body text), semantic HTML, keyboard-navigable controls, reduced motion support via `prefers-reduced-motion`. No strict certification target.
